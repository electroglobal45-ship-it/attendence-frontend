║                           VIEW SWITCHING SYSTEM                               ║
╚══════════════════════════════════════════════════════════════════════════════╗

HOW IT WORKS:
1. Click "Board" button in header (shows current view name)
2. Dropdown opens with all 6 view options
3. Click any view to switch
4. View changes instantly
5. Header button updates to show current view name

STATE MANAGEMENT:
  const [currentView, setCurrentView] = useState<'board'|'table'|'timeline'|'calendar'|'dashboard'|'map'>('board')

VIEW RENDERING:
  {currentView==='board' && <TrelloBoard .../>}
  {currentView==='table' && <TableView .../>}
  {currentView==='timeline' && <TimelineView .../>}
  {currentView==='calendar' && <CalendarView .../>}
  {currentView==='dashboard' && <DashboardView .../>}
  {currentView==='map' && <MapView .../>}

╔══════════════════════════════════════════════════════════════════════════════╗
║                           TECHNICAL DETAILS                                   ║
╚══════════════════════════════════════════════════════════════════════════════╗

COMPONENT STRUCTURE:

KanbanView (Main)
├─ State: currentView
├─ Header with view dropdown
└─ Content area renders:
    ├─ TrelloBoard (board view)
    ├─ TableView (table view)
    ├─ TimelineView (timeline view)
    ├─ CalendarView (calendar view)
    ├─ DashboardView (dashboard view)
    └─ MapView (map view)

PROPS PASSED TO VIEWS:
  boardData: {
    board: Board
    lists: List[]
    tasks: Task[]
    labels: any[]
    members: any[]
  }
  onTaskClick: (task: Task) => void

DATA FLOW:
1. KanbanView loads board data
2. Stores in boardData state
3. Passes to current view component
4. View renders data in specific format
5. Click handlers bubble up to KanbanView
6. Task detail modal opens
