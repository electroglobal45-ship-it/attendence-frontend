═══════════════════════════════════════════════════════════════
  HOW IT WORKS NOW
═══════════════════════════════════════════════════════════════

CREATE LABEL:
1. Click "Create New Label" in TaskDetailModal
2. Enter name and pick color
3. POST /api/v1/labels → Creates in board_labels table
4. Label immediately available for all tasks in that board

ASSIGN LABEL:
1. Click label picker in TaskDetailModal
2. Select a label
3. PUT /api/v1/tasks/:id with labels: [{ id, name, color }]
4. Backend:
   - Verifies label exists in board_labels
   - Verifies label belongs to correct board
   - Deletes old task_board_labels entries
   - Creates new task_board_labels entry
5. Label shows on card

DISPLAY LABEL:
- Card.tsx: Shows label.color and label.name from task.labels[0]
- No fallback to hardcoded priorities
- If no label, shows nothing (no default)

═══════════════════════════════════════════════════════════════
  API ENDPOINTS
═══════════════════════════════════════════════════════════════

GET    /api/v1/labels/boards/:boardId
       → Returns all labels for a board

POST   /api/v1/labels
       Body: { board_id, name, color }
       → Creates new label

PUT    /api/v1/labels/:labelId
       Body: { name?, color? }
       → Updates label

DELETE /api/v1/labels/:labelId
       → Deletes label

PUT    /api/v1/tasks/:taskId
       Body: { labels: [{ id, name, color }] }
       → Assigns label to task
