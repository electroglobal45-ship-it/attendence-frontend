TESTING GUIDE - OPTIMIZED KANBAN & ATTENDANCE
==============================================

SETUP INSTRUCTIONS
==================

1. Start Backend Server:
   - Open terminal in project root
   - Run: cd backend
   - Run: npm run dev
   - Backend should start on http://localhost:5000

2. Start Frontend:
   - Open new terminal in project root
   - Run: npm run dev
   - Frontend should start on http://localhost:3000

3. Login:
   - Admin: Use admin credentials
   - Employee: Use employee credentials

KANBAN TESTS
============

Test 1: Card Movement Without Reload
-------------------------------------
Steps:
1. Login as admin
2. Go to Tasks → Click "Kanban" button
3. Drag a card from "To Do" to "In Progress"
4. Expected: Card moves instantly, no page reload
5. Check: "Moving card..." indicator appears briefly
6. Verify: Card stays in new position after indicator disappears

Test 2: Add List Without Reload
--------------------------------
Steps:
1. On Kanban board, click "Add List" button
2. Enter list name (e.g., "Testing")
3. Click "Add List"
4. Expected: New list appears immediately, no reload
5. Verify: Can drag cards to new list

Test 3: Update Task Details
----------------------------
Steps:
1. Click on any card to open detail modal
2. Click "Edit" on description
3. Type some text in the rich text editor
4. Click "Save"
5. Expected: Modal stays open, no flickering
6. Close modal and reopen same card
7. Verify: Changes are saved

Test 4: Add Checklist Item
---------------------------
Steps:
1. Open a card
2. Click "+ Add checklist item"
3. Type item name and press Enter
4. Expected: Item appears immediately
5. Click checkbox to mark complete
6. Expected: Instant visual feedback
7. Verify: Progress bar updates immediately

Test 5: Admin Real-Time Updates
--------------------------------
Steps:
1. Open two browsers (or incognito)
2. Login as admin in Browser 1
3. Login as employee in Browser 2
4. In Browser 2: Move a card
5. Wait 30 seconds
6. In Browser 1: Card should update automatically
7. Verify: No manual refresh needed

COMMON ISSUES & FIXES
=====================

Issue: Cards snap back after moving
Fix: Check backend console for errors, verify API endpoint works

Issue: Polling doesn't work
Fix: Check browser console, verify 30-second interval is running

Issue: Camera doesn't open
Fix: Check browser permissions, HTTPS required for camera on mobile

Issue: GPS timeout
Fix: Enable location services, try outdoor for better signal

Issue: Backend not starting
Fix: Check .env file exists, verify Supabase credentials

Issue: CORS errors
Fix: Verify CORS_ORIGIN in backend .env matches frontend URL

BROWSER CONSOLE CHECKS
======================

Good Signs:
- No red errors
- "Moving card..." logs (optional)
- Network requests complete successfully (200 status)

Bad Signs:
- 401 Unauthorized (check auth token)
- 404 Not Found (check API endpoints)
- CORS errors (check backend CORS config)
- Network errors (check backend is running)

SUCCESS CRITERIA
================

✓ Card movements feel instant
✓ No full page reloads on any action
✓ Modal stays open during updates
✓ Admin sees changes within 30 seconds
✓ Attendance page shows structure immediately
✓ All CRUD operations work without flickering
✓ Error states handled gracefully
✓ Mobile experience is smooth

PERFORMANCE METRICS
===================

Target Response Times:
- Card movement: <100ms (UI update)
- Add list: <100ms (UI update)
- Task detail update: <200ms
- Page load: <2s
- Backend sync: <500ms

Memory Usage:
- Should not increase over time
- No memory leaks from polling
- Proper cleanup on component unmount
