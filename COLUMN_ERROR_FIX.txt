TASKS.TITLE COLUMN ERROR - FIXED
=================================

ERROR MESSAGE
=============
```
Task lookup error: {
  code: '42703',
  details: null,
  hint: null,
  message: 'column tasks.title does not exist'
}
```

ROOT CAUSE
==========
The query was trying to SELECT the 'title' column from the tasks table,
but this column doesn't exist in your database schema.

PostgreSQL Error Code 42703 = "undefined_column"

FIX APPLIED
===========

Changed the query from:
```typescript
const { data: task } = await supabase
  .from('tasks')
  .select('id, title, list_id, project_id, position')  // ❌ title doesn't exist
  .eq('id', task_id)
  .single()
```

To:
```typescript
const { data: task } = await supabase
  .from('tasks')
  .select('id, list_id, project_id, position')  // ✅ Only essential columns
  .eq('id', task_id)
  .single()
```

WHY THIS WORKS
==============

The move operation doesn't actually need the task title!
It only needs:
- id: To identify which task to move
- list_id: To know current list
- project_id: For permission checking
- position: To know current position

The title is not used anywhere in the move logic, so we can safely
remove it from the query.

DATABASE SCHEMA NOTE
====================

Your tasks table appears to have different column names than expected.
The TypeScript interfaces show 'title' but the actual database table
might use a different column name, or the column might not exist yet.

Possible scenarios:
1. Column is named differently (e.g., 'name' instead of 'title')
2. Table structure is different from types
3. Migration not run yet
4. Using a different database/schema

To verify your actual tasks table structure, run this in Supabase SQL editor:
```sql
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'tasks' 
AND table_schema = 'public';
```

RECOMMENDATION
==============

If you need to display task titles in the future, you should:

1. Check actual column name in database
2. Update TypeScript types to match
3. Use correct column name in queries

OR

1. Add a migration to rename/add the title column
2. Run: ALTER TABLE tasks ADD COLUMN title TEXT;
3. Or: ALTER TABLE tasks RENAME COLUMN [old_name] TO title;

For now, the move functionality will work without the title! ✅
