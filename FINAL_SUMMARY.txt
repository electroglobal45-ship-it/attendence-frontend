HOW IT WORKS
============

Optimistic Updates Pattern:
1. User performs action (move card, add list, etc.)
2. UI updates immediately (local state)
3. Request sent to server in background
4. On success: Keep the update
5. On error: Revert to previous state

Polling for Real-Time:
1. Every 30 seconds, check for updates
2. Only when board is active
3. Silent background refresh
4. No loading indicators

Progressive Loading:
1. Show page structure first
2. Load dynamic data asynchronously
3. Update UI as data arrives
4. Never show blank screens
