HOW IT WORKS
===============================================================================

CHECK-IN FLOW:
1. User opens attendance page
2. System fetches holidays from database
3. System checks if today is blocked:
   - Sunday? → Show "Sunday - Weekly off"
   - 3rd Saturday? → Show "3rd Saturday - Monthly off"
   - Holiday? → Show "Public Holiday - [Name]"
4. If blocked:
   - Show blue info card with reason
   - Hide check-in form
   - Show "Enjoy your day off! 🎉"
5. If allowed:
   - Show normal check-in form
   - Require selfie + GPS
   - Submit attendance

CHECKOUT FLOW:
1. User clicks "Start Checkout Process"
2. Enters checkout mode
3. Must capture selfie
4. Must enable GPS
5. Both required before submission
6. Click "Check Out" button
7. Data uploaded and stored

API VALIDATION:
1. Request received at /api/attendance/mark
2. Fetch holidays from database
3. Check if date is blocked
4. If blocked → Return 400 error with reason
5. If allowed → Process attendance

- check_out: TIMESTAMP
